# 旅游社交项目部署手册

## 1. 项目信息

项目由后端 Spring Boot 服务和前端 UniApp 应用组成。

| 项目 | 目录/名称 | 说明 |
| --- | --- | --- |
| 后端 | `springboot-travel-socail` | Spring Boot 后端服务 |

| Redis | `redis-server` | 缓存服务，后端启动前需要先启动 |
| 访问地址 | `http://116.62.59.134:99/` | 前端和接口访问地址 |
| 宝塔面板 | `http://116.62.59.134:8888/d96c444c` | 服务器管理面板 |

> 注意：宝塔面板账号、密码属于敏感信息。部署时使用项目提供的账号 `s5flmixi` 和对应密码登录，部署文档不要在公网或代码仓库中公开明文密码。

## 2. 部署前准备

本地或服务器需要准备以下环境：

1. JDK 8 或项目要求的 JDK 版本。
2. Maven 3.x。
3. Redis。
4. HBuilderX。
5. 宝塔面板。
6. MySQL 或项目配置文件中指定的数据库环境。

建议先确认端口放行：

| 端口 | 用途 |
| --- | --- |
| `99` | 项目前端/后端访问端口 |
| `8888` | 宝塔面板端口 |
| `6379` | Redis 默认端口，仅建议服务器内网访问 |
| `3306` | MySQL 默认端口，仅建议服务器内网访问 |

## 4. 安装并启动 Redis

### 4.1 Windows 本地启动方式

如果是在 Windows 环境运行 Redis：

1. 下载 Redis Windows 版本。
2. 解压 Redis。
3. 进入 Redis 解压目录。
4. 执行：

   ```powershell
   .\redis-server.exe redis.windows.conf
   ```

如果没有 `redis.windows.conf`，可以直接执行：

```powershell
.\redis-server.exe
```

### 4.2 Linux 服务器启动方式

如果是在 Linux 服务器运行 Redis：

```bash
redis-server
```

后台运行可以使用：

```bash
nohup redis-server > redis.log 2>&1 &
```

验证 Redis 是否启动：

```bash
redis-cli ping
```

返回：

```text
PONG
```

表示 Redis 启动成功。

## 5. 后端部署：springboot-travel-socail

### 5.1 修改后端配置

进入后端项目：

```bash
cd springboot-travel-socail
```

检查配置文件，常见位置如下：

```text
src/main/resources/application.yml
src/main/resources/application-dev.yml
src/main/resources/application-prod.yml
src/main/resources/application.properties
```

重点确认以下配置：

```yaml
server:
  port: 99

spring:
  redis:
    host: 127.0.0.1
    port: 6379
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/数据库名?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: 数据库用户名
    password: 数据库密码
```

如果项目使用的是前后端分离，并且前端直接访问后端接口，后端端口也可以按项目实际配置设置。本文按访问地址 `http://116.62.59.134:99/` 编写。

### 5.2 打包后端

在后端项目根目录执行：

```bash
mvn clean package -DskipTests
```

打包成功后，通常会在 `target` 目录生成 `.jar` 文件，例如：

```text
target/springboot-travel-socail.jar
```

实际文件名以项目打包结果为准。

### 5.3 启动后端

进入 jar 所在目录后执行：

```bash
java -jar springboot-travel-socail.jar
```

后台启动方式：

```bash
nohup java -jar springboot-travel-socail.jar > app.log 2>&1 &
```

查看日志：

```bash
tail -f app.log
```

验证后端是否启动：

```bash
curl http://116.62.59.134:99/
```

如果项目有健康检查接口，优先访问健康检查接口，例如：

```bash
curl http://116.62.59.134:99/actuator/health
```

## 6. 前端部署：uniapp-travel-socail

### 6.1 使用 HBuilderX 打开项目

1. 打开 HBuilderX。
2. 选择 `文件` -> `导入` -> `从本地目录导入`。
3. 选择前端项目目录：

   ```text
   uniapp-travel-socail
   ```

4. 等待 HBuilderX 识别 UniApp 项目。

### 6.2 修改前端接口地址

前端需要把请求地址统一改为：

```text
http://116.62.59.134:99/
```

建议优先搜索以下关键词：

```text
baseUrl
BASE_URL
apiUrl
request
localhost
127.0.0.1
http://
https://
```

常见文件位置：

```text
uniapp-travel-socail/common/request.js
uniapp-travel-socail/utils/request.js
uniapp-travel-socail/config/index.js
uniapp-travel-socail/api/index.js
uniapp-travel-socail/main.js
```

按实际项目文件为准。

示例修改：

```js
// 服务器部署地址：后端服务访问地址，部署到 116.62.59.134 时使用
const BASE_URL = 'http://116.62.59.134:99/'
```

如果原来是：

```js
const BASE_URL = 'http://localhost:8080/'
```

修改为：

```js
// 部署地址：统一指向服务器后端服务
const BASE_URL = 'http://116.62.59.134:99/'
```

如果项目使用 `uni.request`，示例：

```js
uni.request({
  // 部署地址：接口请求统一访问服务器 99 端口
  url: 'http://116.62.59.134:99/api/user/login',
  method: 'POST',
  data: formData,
  success: (res) => {
    console.log(res)
  }
})
```

更推荐把地址抽成统一配置，避免多个页面重复修改：

```js
// 部署地址：服务器后端服务根地址
export const BASE_URL = 'http://116.62.59.134:99/'
```

然后请求时统一引用：

```js
import { BASE_URL } from '@/config/index.js'

uni.request({
  url: BASE_URL + 'api/user/login',
  method: 'POST',
  data: formData
})
```

## 7. 使用宝塔部署前端静态资源

如果前端需要打包成 H5 并部署到服务器：

1. 在 HBuilderX 中选择：

   ```text
   发行 -> 网站-PC Web 或手机 H5
   ```

2. 打包完成后，找到生成的 `dist/build/h5` 目录。
3. 登录宝塔面板。
4. 创建或选择站点。
5. 将 `dist/build/h5` 中的文件上传到站点根目录。
6. 确认站点绑定端口或反向代理到：

   ```text
   http://116.62.59.134:99/
   ```

如果 `99` 端口已经由后端 Spring Boot 占用，则需要明确部署结构：

- 方案一：Spring Boot 直接提供接口，前端 H5 使用其他端口或 Nginx 站点访问。
- 方案二：Nginx 使用 `99` 端口，前端静态资源由 Nginx 提供，接口反向代理到 Spring Boot 的内部端口。

推荐生产环境使用方案二，例如：

```nginx
server {
    listen 99;
    server_name 116.62.59.134;

    root /www/wwwroot/uniapp-travel-socail;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

如果使用该方案，后端 Spring Boot 建议运行在 `8080`，前端访问地址仍然配置为：

```js
// 部署地址：Nginx 对外地址，由 Nginx 转发 /api 请求到后端
const BASE_URL = 'http://116.62.59.134:99/'
```

## 10. 常见问题

### 10.1 页面打不开

检查：

1. 宝塔安全组是否放行 `99` 端口。
2. 云服务器安全组是否放行 `99` 端口。
3. 后端或 Nginx 是否已启动。
4. 防火墙是否拦截。

### 10.2 接口请求失败

检查：

1. 前端地址是否仍然是 `localhost` 或 `127.0.0.1`。
2. 是否遗漏某些页面中的硬编码地址。
3. 后端是否允许跨域。
4. 后端实际接口路径是否和前端请求路径一致。

### 10.3 Redis 连接失败

检查：

1. Redis 是否启动。
2. Redis 地址和端口是否正确。
3. 如果 Redis 设置了密码，后端配置中是否填写了密码。
4. 服务器防火墙是否阻止本机连接。

### 10.4 端口冲突

如果 `99` 端口已被占用：

1. 查看占用进程。
2. 停止旧进程，或修改后端/Nginx 端口。
3. 重新启动服务。

Linux：

```bash
lsof -i:99
```

Windows：

```powershell
netstat -ano | findstr :99
```

## 11. 推荐生产部署结构

生产环境建议使用：

```text
用户浏览器
   |
   v
http://116.62.59.134:99/
   |
   v
Nginx / 宝塔站点
   |-- 前端静态资源：uniapp-travel-socail 打包后的 H5 文件
   |
   |-- /api/ 反向代理
          |
          v
Spring Boot 后端：springboot-travel-socail
          |
          v
Redis / MySQL
```

这种方式便于统一域名和端口，也便于后续配置 HTTPS、日志和反向代理。
